# Rules: Knowledge Base Update

> Context: Keep the project's brain up-to-date. Documentation is part of the deliverable.

## 1. Update Triggers

* **MUST Update:** If there are changes to behavior, public APIs, configurations, known pitfalls, troubleshooting steps, or architecture decisions.
* **SKIP Update:** Internal refactoring or private logic changes (unless they directly affect other maintainers).

## 2. Target Locations

* Apply updates to the relevant files within `.ai\context\`:
  * `FAQ`
  * `Troubleshooting Guide`
  * `Dev Guide`
  * `Architecture Notes`
  * `CHANGELOG`
